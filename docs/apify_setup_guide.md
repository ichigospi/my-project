# Apify契約 → Threadsツール接続 手順書（オーナー向け）

スクレイパー（競合の自動収集・自投稿の自動計測）に使う Apify の契約手順です。所要10〜15分。

## 大前提（BAN回避）

- **運用アカウントのログイン情報・Cookieは絶対に入力しない**。Apifyは業者側のインフラで公開投稿を収集するため、この分離を守る限り運用アカウントに紐づくリスクは実質ありません。
- 登録に使うメールは運用アカウントと無関係なものでOK（会社のメールで可）。

## 手順

### 1. アカウント作成（無料）
1. https://apify.com にアクセス → 右上 **Sign up**
2. メールアドレスで登録（Googleログインでも可）
3. 登録直後は無料プラン（月$5分の無料クレジット付き）。**まずは無料のまま接続テストまで進めてOK**

### 2. プラン契約（テスト後でよい）
1. ダッシュボード左下 **Billing** → **Subscriptions**
2. **Starter（月$39）** を選択 — 競合5〜10アカウントの日次収集＋自投稿計測ならこれで足ります
3. 支払いカードを登録

### 3. APIトークンの発行
1. 右上のアカウントアイコン → **Settings**
2. 左メニュー **API & Integrations**
3. **Personal API tokens** にあるトークンをコピー（なければ **+ Create a new token**）
   - `apify_api_` で始まる文字列です

### 4. ツールに登録
1. Threadsツールの **設定** 画面（/threads/settings）を開く
2. 「スクレイパー（Apify）」の **APIトークン** に貼り付けて保存
3. Actor ID は空欄のまま（接続実装時にこちらで最適なActorを設定します）

### 5. 完了報告
トークン保存まで終わったら「Apify入れたよ」と一言ください。こちらで接続実装（競合自動収集＋自動計測cron）に着手します。

## 費用の目安

| 項目 | 金額 |
|---|---|
| Starterプラン | $39/月（約6,000円） |
| 収集の従量（競合10アカ日次+自投稿計測） | プラン内クレジットでおおむね収まる想定。超過時は使った分だけ |

※ まず無料クレジットで1〜2週間試し、実際の消費量を見てから契約判断でもOKです。
