# 外部連携の設定手順（UTAGE / BASE）

このツールの「設定」ページ最下部に連携セクションがあります。以下の手順で繋ぐと手入力がほぼ不要になります。

---

## 1. UTAGE連携（リストイン・無料鑑定の自動記録）

UTAGEのシナリオやファネルの「アクション」機能で、登録・申込のタイミングにwebhook送信を追加します。

### 手順

1. このツールの **設定 → UTAGE連携** で Webhook URL とアカウントID（恋愛/金運それぞれ）を確認・コピー
2. UTAGE管理画面で対象のシナリオ（例: LINE登録シナリオ）を開く
3. 登録直後のステップに **アクション → webhook送信（外部連携）** を追加
4. URLに以下を設定（`<>` 部分を置き換え）:

```
https://<このツールのURL>/api/ingest/utage?key=<シークレット>&event=list_in&account=<アカウントID>&source=<媒体>
```

- `event` の値:
  - `list_in` = リストイン（LINE登録）
  - `free_apply` = 無料鑑定の申込（申込フォームのシナリオに設定）
  - `free_sent` = 鑑定送付（UTAGEで送付を自動化している場合）
- `source` の値: `threads` / `insta` / `x` / `youtube` / `other`
  - **媒体別に登録シナリオ（または流入経路別の登録URL）を分けている場合**、それぞれのシナリオに `source` だけ変えた同じwebhookを設定すると、媒体別リストインが自動で計測されます
- POSTボディに `email` や `name` を含めると読者単位でも記録されます（UTAGEの差し込みコードを利用。任意）

### 確認方法

設定後、テスト登録を1件行い、このツールのダッシュボード（今日）でリストインが1増えていればOKです。

### 注意

- URLに含まれる `key=` はパスワードに相当します。外部に公開しないでください
- 過去分のデータはwebhookでは入りません。UTAGEの読者CSVをエクスポートして取り込みたい場合は相談してください（実績入力からの手入力でも可）

## 2. BASE連携（売上の自動計上）

BASE APIで注文情報を取得し、売上として自動で取り込みます。

### 手順

1. [BASE Developers](https://developers.thebase.com/) にBASEアカウントでログインし、アプリを新規登録
   - コールバックURL: このツールの **設定 → BASE連携** に表示されている「コールバックURL」をそのまま指定
   - 利用するAPI（スコープ）: **read_orders（注文情報の参照）** にチェック
2. 発行された `client_id` と `client_secret` を設定画面に入力して **設定を保存**
3. **同期先アカウント**（恋愛/金運のどちらのショップか）と**デフォルトカテゴリ**を選択
4. **商品名→カテゴリ判定ルール** を設定（例: 商品名に「継続」を含む → アップセル、「講座」→ 講座）
5. **BASEと接続** を押してBASEの認可画面で許可
6. **今すぐ同期** を押して注文が取り込まれることを確認

### 同期の仕組み

- ダッシュボードを開いたとき、前回同期から1時間以上経っていれば自動で同期されます
- 手動の「今すぐ同期」もいつでも可能
- 注文IDで重複チェックしているので、何度同期しても二重計上はされません
- 取り込まれた売上は売上内訳・ダッシュボード・テンプレ成績（手動で紐付けた場合）に反映されます

### 注意

- BASEのaccess tokenは1時間で失効しますが、自動で更新されます。refresh tokenも失効した場合（30日以上未使用など）は「BASEと接続」をやり直してください
- ショップが2つ（恋愛/金運で別ショップ）ある場合は、現状は主要ショップ1つを接続してください。2ショップ対応が必要になったら拡張します
