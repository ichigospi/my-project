## インセンティブ設計とは

> **読者が自分から「〇〇しなきゃ！」と感じるような動機・刺激を、各アクションに仕掛けておくこと。**

### マスターするとできること

- 狙ったポストにいいねを大量につける
- 読者が **通知オン** して待機する状態を意図的に作る
- スペース・無料プレゼント企画で初速参加者を圧倒的に増やす
- ローンチ時に伏線回収で **最大火力** を出す
- ポスト直後に30〜50いいねがつく異常スピード

→ **思いのままに読者を動かす** ことが可能になる。
→ SNSで物を売るのが **イージー化** する。

### 教材執筆者の異常な数字

- ポストして **数分後に30〜50いいね** はザラ
- 本気のポストは **10〜20万インプ** を超える
- 通常のアルゴ理論では説明できない初速

→ これは **インセンティブ設計** によって作られている。

## 大前提: 「ファン0人スタンス」

> 出典: 教材8（2026-04-27）

### ⚠️ 楽観運用は3流

ある程度発信が伸びてくると、

- 「どうせいいねくれるっしょ」
- 「ポストしたら見てくれるっしょ」
- 「スペース軽く告知したら集まるっしょ」
- 「商品も買ってくれるっしょ」

という **楽観スタンス** で運用してしまいがち。

→ **これがXで商品が売れなくなる原因**。
→ こういう運用は **3流**。

### 自分にファンは0人スタンスで運用する

教材執筆者の運用スタンス:

> 「自分にファンは1人もいない」というスタンスで、Xを運用している。

#### なぜか

- お客さんは **あなたのファンではない**
- 本人は「ファンです」と言うかもしれないが…
- それは **自分にとって都合のいい発信** をしてくれるからフォローしてるだけ
- 別に得がなければ **基本は行動してくれない**
- 無条件にいいね・リプ・引用などしてくれるわけではない

→ この現実を理解した上で運用する。

### 中核原則

```
人間は自分にメリットがないと行動しない。
```

→ 他人に何かしてほしいなら、必ず **インセンティブを設計する**。
→ 「〇〇してください」と直接お願いするだけでは弱い。

## インセンティブの3類型

> インセンティブ＝何か行動したくなるような動機や刺激のこと。

```
① AをしたらBがもらえる    （特典・報酬型）
② Aをしたら認知してもらえる （承認型）
③ Aをしたら損を回避できる  （損失回避型）
```

### 例

| 行動 | 特典型 | 承認型 | 損失回避型 |
|---|---|---|---|
| いいね | いいね100で続編公開 | いいね多い人を褒める | いいねしないと続き読めない |
| 引用RT | 拡散者特典 | 拡散者を紹介 | 拡散しないと外される |
| 通知オン | 通知オン勢に限定情報 | 通知オン勢を「なまチル勢」と命名 | 通知オフだと見逃して後悔 |
| 感想記入 | 感想特典 | 感想を本人にいいねされる | 感想ないと商品買えない |
| スペース参加 | 参加者限定資料 | 参加者を呼名 | 参加しないと聞き逃す |

## 行動別インセンティブの実装

### 通知オン化のインセンティブ（最重要）

ポスト直後の異常な反応速度の **正体は通知オン**。
通知オン勢が常時待機しているからこそ、ポスト直後に反応が集中する。

#### テクニック: ポスト即消し戦略

レアな情報をポストして **即消し** する運用。

##### 機能する仕組み

```
ポスト即消し
  ↓
通知オン勢「見れた！」
通知オフ勢「えー！見れなかった！」
  ↓
通知オフ勢が後悔・好奇心を感じる
  ↓
「次は見逃さないように」通知オンする
```

##### 実例ポスト

```
✅「ポスト消した。通知オンしてポスト見れた人はラッキー」
✅「久々にポスト消した。通知オンだったなまチル勢は見れたよね」
✅「ポストを1つ消した。悪いポストは即削除する。
   見れた人はラッキー」
```

##### 機能した結果のリプ例

- 「通知オンで見れました！！」
- 「通知オンしてたけど見れませんでした…」
- 「通知オンにしました！」

→ こうしたリプを見た他の読者も「自分も通知オンせざるを得ない」状態になる（追従効果）。

##### 重要原則

通知オンを **直接お願いしない**。
「通知オンしてください」と言うのではなく、**通知オンしないと後悔する状況** を作る。

### 感想記入のインセンティブ

「コンテンツの感想書いて！」と直接お願いしても **読者はあまり書かない**。

#### テクニック: 感想特典

```
「感想書いてくれた方に〇〇プレゼント」
```

これでようやく感想が回り出す。

#### 注意

感想特典は多くの人がやっているが、その **土台にあるインセンティブの考え方** をマスターしてる人は少ない。
→ 表面パクリではなく、原理を理解して他のアクションにも応用する。

### いいね・引用RTのインセンティブ

- 「いいね100で続編公開」（特典型）
- 「いいね多い人を褒める」（承認型）
- 「拡散してくれた方の中から選んで紹介」（承認型）

## 伏線としてのインセンティブ

> インセンティブは **単発のポスト** だけでなく、**ポスト同士の連鎖** でも作られる。

### きあいだめ理論

ポケモンの「きあいだめ」が次の技で急所に当たりやすくなるように、
**単体では意味がないジャブポスト** を入れることで、**次のポストの威力を上げる**。

```
[ポスト A] ジャブ（単体では意味がない）
  ↓
読者の注目度が上がる
  ↓
[ポスト B] 本命のメッセージ（高威力で着弾）
```

### ジャブポストの例

```
✅「最悪な出来事が起こりました、、、」
✅「これまでの努力が無駄になったんやが、、、」
✅「ヤバいこと起こった...」
✅「ショックすぎて気絶してました...夜にまた話します」
```

→ これら単体は教育もセールスもしていない。
→ ただし **次のポストへの注目を圧倒的に高める**。

### 機能する仕組み

```
ジャブポスト「最悪な出来事…」
  ↓
読者「え、何があったの？！」
  ↓
次のポストを見逃さないように待つ
  ↓
本命の教育/セールスポストが高注目度で着弾
```

### 「次の威力を上げる」という考え方

```
❌ 1ポストごとに完結させる
⭕ 前のポストが次のポストの威力を上げる連鎖を作る
```

→ ストーリー型運用とインセンティブ設計は **相互強化** する関係。

## 商品ローンチへの伏線回収設計

> 出典: 教材8（2026-04-27）

### 普通の運用（火力が出ない）

```
コンテンツ作成教材を売りたい
  ↓
「コンテンツが資産」「ストック収入」「集客装置」等の教育ポストを連発
  ↓
教育ポストが伸びない（インプが出ない）
  ↓
読者が見てくれない
  ↓
威力が半減
```

→ **教育がうまくても、見てもらえなければ意味がない**。

### なまいきくん式（最大火力）

```
ローンチ前から、各アクションへの伏線を貼る:
  - 「ポストを見る」インセンティブ
  - 「スペースに参加する」インセンティブ
  - 「無料プレゼントを受け取る」インセンティブ
  - 「通知オンする」インセンティブ
  ↓
読者が能動的に発信を追う状態を作る
  ↓
商品リリースのタイミングで一気に伏線回収
  ↓
最大火力で売れる
```

### 設計の基本パターン

ローンチを意識した時、**何ヶ月も前から** 各アクションのインセンティブを仕込む:

| アクション | 仕込むインセンティブの例 |
|---|---|
| 通知オン | 即消しポストを定期的にやる |
| 感想記入 | 過去のコンテンツで感想特典をやり、特典を強化 |
| スペース参加 | 過去のスペースで参加者限定情報を出す |
| LINE登録 | LINEだけで配信する情報を出す |
| 引用RT | 拡散者を毎回褒める文化を作る |

→ ローンチ時にこれらが **同時発火** することで、教育ポストの威力が10倍になる。

## のんびりくん裏垢ローンチでの伏線設計（実例）

> 教材6/7のストーリー型運用事例を、インセンティブ視点で再分析。

### 全体ストーリー

```
[1] なまいきくんが裏垢に挑戦する
       ↓
[2] なまいきくんが勘違いして暴走 → エロ画像を撮りまくる
       ↓
[3] 事件が起こる ← 伏線（インセンティブ）として機能
       ↓
[4] 事件の内容が明かされる → エロ画像が不要だとわかる
       ↓
[5] のんびりくんのセミナーで詳しく解説
```

### 「事件」パートのジャブポスト

```
ポスト例:
  「最悪な出来事が起こりました、、、」
  「これまでの努力が無駄になったんやが、、、」
```

これらは **教育もセールスもしていない**。
ただし **次のポスト（エロ画像不要の教育ポスト）への注目を爆上げ** している。

### きあいだめ理論の典型例

```
ジャブポスト「最悪な出来事…」← きあいだめ
  ↓
読者「え、何があったの？！」← 注目度MAX
  ↓
教育ポスト「実はエロ画像は要らなかった」← 急所ヒット
  ↓
セミナー誘導「のんびりくんのセミナーで詳しく」← セールス着弾
```

→ 全体が伏線回収の連鎖になっている。
→ ローンチ時の火力が最大化される。

## 重要な思考パターン

### 「どうやったら〇〇したくなるか？」を常に考える

普通の人:
```
通知オンしてほしい
  ↓
「通知オンしてください」と直接お願いする（or 何もしない）
```

なまいきくん式:
```
通知オンしてほしい
  ↓
「どうやったら通知オンしたくなるか？」を考える
  ↓
ポスト即消し戦略を実装
```

→ **直接依頼ではなく、自然に行動したくなる仕掛け** を考えるクセをつける。

### 全アクションに対してこの視点を持つ

```
いいねしてほしい → どうやったらいいねしたくなるか？
引用RTしてほしい → どうやったら引用RTしたくなるか？
ポストを追ってほしい → どうやったら追いたくなるか？
商品をアフィしてほしい → どうやったらアフィしたくなるか？
感想を書いてほしい → どうやったら書きたくなるか？
通知オンしてほしい → どうやったら通知オンしたくなるか？
```

各アクションに対して必ず **インセンティブ設計** を考える。
