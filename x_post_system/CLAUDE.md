# Xポスト作成システム — 全体方針

ビジネス系・占いスピ系アカウントの **Xポスト作成 / 競合分析 / 自アカ転用** を行うシステム。
ノウハウは本フォルダに蓄積し、Webアプリ (`/x-post`) からも参照する「司令塔」として機能する。

## トリガー

ユーザーから以下の指示があった場合、本システムを使用する:

- 「Xポストを作って」「ツイートを作って」
- 「（競合の）伸びてる投稿を分析して」
- 「ビジ系（or 占い系）のポストを作って」
- 「参考ポストに追加して」「教材を読み込ませて」「自アカ情報を更新して」

## フォルダ構成

```
x_post_system/
├── CLAUDE.md                      ← このファイル（全体方針・AI指示）
├── knowledge_common/              ← 両ジャンル共通ノウハウ
│   ├── post_principles.md         # ポスト作成の原則（共通）
│   ├── structure_patterns.md      # 構成パターン（フック→展開→締め 等）
│   └── ng_patterns.md             # NGパターン
├── knowledge_business/            ← ビジネス系専用
│   ├── account_info.md            # 自アカ情報（コンセプト・ターゲット等）
│   ├── teachings.md               # 教材から抽出した指示書
│   └── reference_posts.md         # 参考ポスト集（伸びた投稿のお手本）
├── knowledge_spiritual/           ← 占いスピ系専用
│   ├── account_info.md
│   ├── teachings.md
│   └── reference_posts.md
├── prompts/                       ← アプリが参照するプロンプト
│   ├── analyze_trending.md        # 伸びてる要素分析プロンプト
│   └── generate_post.md           # ポスト生成プロンプト
└── logs/                          ← 生成物・作業ログ保存先
```

## ジャンル定義

| ジャンル | キー | 対象 |
|---|---|---|
| ビジネス系 | `business` | ビジネス・マーケ・副業・自己啓発系アカウント |
| 占いスピ系 | `spiritual` | 占い・スピリチュアル・開運・神社・ヒーリング系アカウント |

両ジャンルに共通する原則・型は `knowledge_common/` に置く。
ジャンル特有のものだけ `knowledge_business/` または `knowledge_spiritual/` に置く。
**迷ったら共通に入れる**（後で重複が出にくい）。

## ノウハウ種別

各ジャンルフォルダ内は3種類に分類:

| 種別 | ファイル | 内容 |
|---|---|---|
| 自アカ情報 | `account_info.md` | コンセプト・ターゲット・口調・USP・避けたい表現など |
| 教材 | `teachings.md` | 学んだ教材・ノウハウから抽出した「やるべきこと/避けるべきこと」 |
| 参考ポスト | `reference_posts.md` | 伸びた投稿の実例集（自他問わず）。構造分析つき |

## ノウハウ追加フロー（PDF / URL / コピペ対応）

### 1. PDF読み込み

ユーザーがPDFをチャットで共有 → 以下を実行:

1. PDFの内容を読む
2. **「共通／ビジ／占い」「自アカ情報／教材／参考ポスト」のどこに入れるか分類提案**
3. 既存ファイルとの重複・矛盾をチェック
4. 構造化Markdown（原則・例・NG・出典）に整形
5. ユーザー確認後、該当ファイルへ追記

### 2. URL読み込み（伸びた投稿）

ユーザーがX投稿のURLを共有 → 以下を実行:

1. 本文・投稿者・指標（推定）を確認
2. ジャンル・構造（フック型・ストーリー型・リスト型 等）を分類
3. 該当する `reference_posts.md` に「実例 + 分析」セットで追記

### 3. 文章コピペ

ユーザーがテキストを直接貼った場合も同様の流れで分類・追記。

## アプリ (`/x-post`) との連携

本フォルダは **アプリのプロンプト源** として機能する:

- アプリの「ポスト分析」機能は `prompts/analyze_trending.md` + `knowledge_common/structure_patterns.md` を参照
- アプリの「ポスト生成」機能は `prompts/generate_post.md` + 該当ジャンルの `account_info.md` + `teachings.md` + `reference_posts.md` を参照
- 競合ポスト収集は **X API v2** で取得し、DBに保存（本フォルダではなくDB管理）

## 共通ルール（生成時）

1. 出力は必ず該当ジャンルの `account_info.md` の口調・温度に従う
2. `teachings.md` の原則を満たす（NG項目に該当しないか確認）
3. `reference_posts.md` の構造を参考にしつつ **コピーはしない**
4. 同じ構文・冒頭フックを連続で使わない
5. 生成物は `logs/output_[日付]_[内容].md` に保存

## コンプライアンス

- 占い系: 「必ず当たる」「全員に効く」等の断定NG（景表法）
- ビジ系: 「絶対稼げる」「保証」等の断定NG
- 個人体験談には「個人の感想」表記
- 詳細は `knowledge_common/ng_patterns.md` 参照
